NonlinearOptimizationTestFunctionsInJulia - Zwischenbericht
Stand: 11. Juli 2025, 10:14 AM CEST

Projektstatus:
- Modular structure: Test functions (rosenbrock.jl, sphere.jl) in src/functions/, each under 100 lines.
- Test function integration: Via src/include_testfunctions.jl.
- TestFunction structure: Refactored for clarity:
  - Fields: f (function), grad (non-in-place gradient), gradient! (in-place gradient), meta (Dict with name, start, min_position, min_value, properties, lb, ub, description).
  - Advantage: KISS-compliant, robust metadata validation.
- Search domains: Implemented in meta[:lb], meta[:ub]:
  - Rosenbrock: [-5, 5]^n (fill(-5.0, n), fill(5.0, n)).
  - Sphere: [-5.12, 5.12]^n (fill(-5.12, n), fill(5.12, n)).
- Gradient wrappers: grad (non-in-place) and gradient! (in-place) for Optim.jl, NLopt, and Hessian methods (Zygote.hessian with ForwardDiff.Dual support).
- Tests: 60/60 tests in test/runtests.jl:
  - Rosenbrock (28): Function values (f([0.5, 0.5]) ~ 6.5), gradients, edge cases (NaN, Inf, 1e-308), properties (multimodal, non-convex), start [0.0, 0.0], minimum [1.0, 1.0].
  - Sphere (28): Function values (f([1.0, 1.0]) ~ 2.0), gradients, properties (unimodal, convex), start [0.0, 0.0], minimum [0.0, 0.0].
  - Filter (4): Tests filter_testfunctions for convex/multimodal functions.
  - Properties: Tests add_property.
- Demos: Five scripts in examples/ (10-15 lines):
  - Optimize_all_functions.jl: Optimizes all functions with L-BFGS, uses tf.gradient!.
  - Compare_optimization_methods.jl: Compares Gradient Descent and L-BFGS on Rosenbrock.
  - List_all_available_test_functions_and_their_properties.jl: Lists functions, start points, minima, properties.
  - Optimize_with_nlopt.jl: Optimizes Rosenbrock with NLopt's LD_LBFGS.
  - Compute_hessian_with_zygote.jl: Shows 3 Newton steps on Rosenbrock, fixed for ForwardDiff.Dual support.
- Load time: Target < 0.5 seconds for two functions, to be optimized with FUNCTION_REGISTRY.
- Compatibility: Julia 1.11.5+, pure Julia code.
- Documentation: Readme.txt (plain text, no Markdown), file comments with path, purpose, context, timestamp.
- Error handling: Replaced @assert with throw(ArgumentError(...)) for robustness.
- Comparison with other packages:
  - Opfunu: 500+ Python benchmark functions, no gradients (internal comparison, not public).
  - CUTEst: Complex installation, our package is Julia-native.
  - Optim.jl, NLopt, Manopt.jl, JuMP/Ipopt, NLSolvers.jl: Compatible, our package adds test functions.

Main Goals:
- Scalability: Via include_testfunctions.jl and planned FUNCTION_REGISTRY.

- Robustness: Precise gradients, stability at [1e-308, 1e-308], robust error handling.
- User-friendliness: Minimal demos (10-15 lines), clear documentation.
